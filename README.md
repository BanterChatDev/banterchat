# Banter

A chat platform built as an alternative to Discord. Free, encrypted, and doesn't ask for your ID.

## What's in here

This repo is the full Banter stack. A Go backend that handles the REST API and WebSocket, and a React frontend that gets bundled and served from the same Go server.

Features that work right now:

- Servers, channels (text and voice), categories, threads
- Direct messages and group DMs
- Friends, blocking, presence (online, idle, do not disturb, invisible)
- Roles with 40+ granular permissions, channel and category permission overrides
- File attachments, custom emojis per server, reactions, mentions
- Slash commands and bots with an OAuth2 invite flow
- Webhooks per channel with rotatable tokens
- Voice and video calls (powered by LiveKit, you bring your own LiveKit server)
- Server discovery directory with tags, language filtering, and bumps
- Site moderation: reports, warnings, suspensions, bans, audit log
- 28 themes and 32 languages
- Encrypted at rest: bios, display names, channel names, message content

## What's not done yet

- macOS and Linux desktop builds (Windows is done)
- Mobile clients
- End-to-end encryption between users (only server-side encryption right now)
- Stickers
- Scheduled events
- Server templates

## Requirements

- Go 1.26 or newer
- PostgreSQL 14 or newer
- Node.js 20 or newer (for the frontend build)
- A LiveKit server if you want voice and video. Run your own with the LiveKit open-source server.

## Setup

```
git clone https://github.com/BanterChatDev/banterchat.git
cd banterchat
./setup.sh
./build.sh
./main
```

`setup.sh` installs Go, Postgres, and Node if they aren't already there. It creates a local Postgres instance in `./pgdata`, generates a fresh `MASTER_KEY`, **prints it on screen and stops** so you can save it somewhere safe, then creates the database, applies the schema, and writes a `.env` file with everything wired up. It is safe to re-run later.

`build.sh` runs `go mod tidy`, installs node modules, builds the frontend bundle, and compiles the Go binary as `./main`.

`./main` starts the server on the address from your `.env` (default `:3030`).

## About the MASTER_KEY

The MASTER_KEY encrypts everything sensitive in your database. Bios, display names, channel names, message content. If you lose it, you cannot decrypt your data. There is no recovery. Save it somewhere outside the server. A password manager works.

`setup.sh` generates it once, prints it on screen, and stops. After you save it, you press enter to keep going.

## Configuration

All configuration lives in `.env`. It's generated by `setup.sh` on first run. If you want to change anything, edit `.env` and restart `./main`. See `.env.example` for the full list of variables and what they do. The important ones:

- `MASTER_KEY` — the encryption key. Generated once by `setup.sh`. Never share it and never commit it.
- `HTTP_ADDR` — what address and port to listen on.
- `DOMAIN` — the domain your instance runs on. Used for cookies and CORS.
- `SECURE_COOKIES` — set to `true` in production behind HTTPS, `false` for local development.
- `LIVEKIT_URL`, `LIVEKIT_API_KEY`, `LIVEKIT_API_SECRET` — LiveKit credentials if you want voice and video. Leave blank to disable.
- `TENOR_API_KEY` — optional Google Tenor key for the GIF picker. Leave blank to disable.
- `TERMS_URL` — the link shown in the warning message sent to users.

## Becoming the site admin

The first user to register on a fresh install becomes the site admin. From there, that admin can promote other people through the admin panel. You can also copy ur user id and place in admin/config.go and restart the server.

## Forking and rebranding

The "Banter" name is hardcoded throughout the frontend (legal pages, warning modal terms link, page titles, brand assets). If you want a hard fork with your own branding, search for `Banter` and `banterchat.org` across the source and replace.

The discover button in the guild selector and the discovery listing links currently point at `guilds.banterchat.org`. If you're running your own instance and want your own discovery directory, edit `components/guilds/GuildSelector.jsx` and `components/discovery/DiscoveryTab.jsx`. If you don't want a discovery feature at all, remove the button.

## Stack

Backend: Go 1.26, Echo, PostgreSQL, gorilla/websocket, LiveKit server SDK.

Frontend: React 18, Tailwind, esbuild. No bundler magic, no Next.js, no SSR. The frontend is one bundled file served by the Go server.
